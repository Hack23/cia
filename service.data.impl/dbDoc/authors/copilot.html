Fix view_riksdagen_party_summary: eliminate fan-out JOIN between view_riksdagen_politician and party_documents (ROOT CAUSE 10). The previous query joined each politician row with all documents for that party (LEFT JOIN party_documents pd ON vp.party = pd.party), creating a massive intermediate result set. All sum(vp.*) fields (total_assignments, total_days_served, etc.) were multiplied by the number of documents per party, producing values in the millions/billions instead of thousands. Fix: Pre-aggregate assignment data from view_riksdagen_politician grouped by party in a CTE, then join with pre-aggregated document data. This eliminates the fan-out and produces correct assignment counts.
